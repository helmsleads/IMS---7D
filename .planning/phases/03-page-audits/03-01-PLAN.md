  <action>
Create `.planning/audit/admin-pages.md` with the document structure defined in 03-RESEARCH.md (overview header, severity legend, source classification legend, feature area sections).

Then audit these feature areas by reading each page source file and evaluating against MASTER.md:

**Dashboard** (~1 page):
- `src/app/(internal)/dashboard/page.tsx`
- Check dashboard widget constraints against DASHBOARD-CONSTRAINTS.md (~360px half-widget minimum)
- Evaluate color tokens, spacing, typography against MASTER.md Sections 1-4

**Inventory** (~5 pages + 2 scanner routes):
- `src/app/(internal)/inventory/page.tsx`
- `src/app/(internal)/inventory/[id]/page.tsx`
- `src/app/(internal)/inventory/pallet-breakdown/page.tsx` (scanner route — apply Section 7 rubric FIRST)
- `src/app/(internal)/inventory/transfers/page.tsx` (scanner route — apply Section 7 rubric FIRST)
- `src/app/(internal)/inventory/import/page.tsx`
- For scanner routes: 44px+ tap targets, 16px+ text, no precision gestures = Blocking violations

**Products** (~3 pages):
- `src/app/(internal)/products/page.tsx`
- `src/app/(internal)/products/[id]/page.tsx`
- `src/app/(internal)/products/categories/page.tsx`

For EACH page, produce a findings table with columns: #, Finding, Lines, Severity, Source, Root Cause, Effort, Current, Recommended.

Classification rules:
- If the visual issue comes from a shared component (Button, Badge, Table, etc.): `source: component` — reference the components.md finding ID (e.g., "RC-01 gray palette — see Badge #3")
- If the page applies its own classes that deviate: `source: inline override`
- If same component defect already documented for a prior page in this audit, use brief reference not full re-documentation

Also check MASTER.md Section 6 anti-patterns: truncated identifiers, relative dates on shipping deadlines, missing units on weight/quantity fields.
