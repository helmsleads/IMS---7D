**For EVERY component (all 27), apply exactly 5 dimensions:**
1. **Visual Alignment** — Colors (gray vs slate, blue vs indigo), shadows (tokenized?), radii (correct step?), transitions (150-200ms, not transition-all on layout props), gradient patterns
2. **Accessibility** — Focus states (focus-visible:ring, not focus:ring), contrast ratios (WCAG AA 4.5:1), ARIA attributes (aria-label on icon-only buttons, roles), alt text, prefers-reduced-motion, keyboard nav
3. **Responsive Behavior** — Mobile rendering, tap target sizes (44px minimum for interactive elements), overflow handling
4. **Variant Completeness** — Missing variants suggested by MASTER.md or industry standard (portal variant for Button/Input/Select, amber instead of yellow for warnings, etc.)
5. **Props API Consistency** — Naming conventions (onClose vs onDismiss), pattern consistency (label/error/hint/required/disabled across form components), className passthrough

**For each finding, record:**
- File path and line range
- Current state (what the code currently does)
- Recommended change (what it should do per MASTER.md)
- Root cause ID (RC-01 through RC-05, or "component-specific" if novel)
- Severity: Blocking / High / Medium / Low (follow RESEARCH.md severity definitions and pitfall guidance)
- Effort estimate: XS / S / M / L

**Pitfall guidance from RESEARCH.md (MUST follow):**
- Gray vs slate: Default to Medium severity. Upgrade to High ONLY if the color is in an interactive/semantic role (e.g., bg-gray-900 on an active page button)
- Token non-use: Default to Medium for brand-critical colors, Low for structural neutrals
- Scanner text-sm: At minimum High. Blocking if it's the primary action identifier text
- StatusBadge: Flag root cause at Badge level. StatusBadge finding should read "inherits Badge color issues — see Badge findings"
- StatCard group-hover:scale-105: Rate Medium (not Blocking) — icon scale is isolated, doesn't cause card layout shift
- Charts: All 14 chart files likely need SVG title/desc + aria-labelledby — rate as High unless proven otherwise
- prefers-reduced-motion gap in globals.css: Document ONCE in a cross-cutting section (Task 2 will add this), reference from animated component findings
