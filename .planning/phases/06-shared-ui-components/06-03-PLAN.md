  <files>src/components/ui/Modal.tsx, src/components/ui/SearchSelect.tsx</files>
  <action>
**Modal.tsx** (Blocking #19, High B-11):
- Add `role="dialog"` and `aria-modal="true"` to the modal container element
- Generate a stable ID for the title (e.g., `useId()` or `modal-title-{id}`) and add `aria-labelledby` pointing to the title element's ID. Add the matching `id` to the title/header element.
- Close button: add `aria-label="Close"` and `focus-visible:ring-2 focus-visible:ring-indigo-500`
- **Focus trap implementation** (use custom approach, no external library):
  - On open: save `document.activeElement` as trigger ref, then focus the first focusable element inside modal (or close button)
  - `onKeyDown` handler on modal container:
    - `Escape`: call onClose
    - `Tab`: get all focusable elements inside modal (`[tabindex]:not([tabindex="-1"]), a[href], button:not(:disabled), input:not(:disabled), select:not(:disabled), textarea:not(:disabled)`). If Shift+Tab on first element, focus last. If Tab on last element, focus first.
  - On close (cleanup): restore focus to saved trigger ref
  - Use `useEffect` to set up focus on mount and restore on unmount

**SearchSelect.tsx** (Blocking #20):
- Add `role="combobox"` to the text input element
- Add `aria-expanded={isOpen}` to input
- Add `aria-controls="listbox-id"` to input (generate stable ID)
- Add `aria-activedescendant` pointing to currently highlighted option ID
- Add `role="listbox"` and `id="listbox-id"` to the dropdown container
- Add `role="option"` and unique `id` to each option item
- Add `aria-selected={isHighlighted}` to each option
- **Keyboard navigation** (if not already present):
  - `ArrowDown`: open dropdown if closed, move highlight to next option
  - `ArrowUp`: move highlight to previous option
  - `Enter`: select highlighted option, close dropdown
  - `Escape`: close dropdown, clear highlight
  - Track `highlightedIndex` state for active descendant
  </action>
